설계 요약 (월주차 포함 확정)

흐름
4자리 입력/OCR → check-collision?include_monthly=true 조회 (debounce 500ms)
• 0건 → "검색 결과 없음" 안내
• 1건 → 카드 없이 즉시 상세로 이동 (권장 · 변경 가능)
• N건 → 카드 리스트 → 선택 → 상세

카드 구분 필드
• 일반차: 4자리 · 차종 · 차색 · 시간 · 위치 · 발렛/자주식
• 월주차: 4자리 · 📅뱃지 · 차종 · 👤차주성함 · 위치 · 발렛/자주식
차주성함·차종은 monthly_parking 계약 조인(customer_name/vehicle_type)으로 자동 표시

구현 범위 (5D)
① check-collision에 include_monthly 파라미터 추가(additive) — 월주차 티켓 포함 + 계약 조인. 5C 입차는 파라미터 미전달 → 무영향.
② 신규 페이지 /v2/crew/exit + 홈 3버튼 + 현황 상단 버튼.
→ 신규 DB·기존 코드 수정 0건 (check-collision는 5B 신규코드라 additive 확장 OK)

별도 파트 (5E) 월주차 입차 4자리화
현재 월주차 입차는 풀번호 자동감지 경로. "월주차 입차도 4자리 + 수기(차종/차주성함)"은 입차 경로·매칭 재설계라 5D와 분리. 5D 출차 검색은 조회만 하므로 지금 월주차 포함 구현에 문제 없음.
