## 🔴 1. P1-8 — v2 입차 차량사진 업로드 (최우선 리스크)

**왜 1순위:** `vehicle-photos` 버킷 정책 SQL이 레포에 없음. CREW(authenticated) 업로드가 RLS로 막히면 사진 기능 자체가 죽음.

| # | 확인 항목 | 기대 결과 |
|---|---|---|
| 1-1 | `/v2/crew/entry`에서 4자리 입차확인 → 사진 단계 진입 | 카메라 1회 오픈, 슬롯 라벨(전면→후면→…) 순차 진행 |
| 1-2 | 1장 촬영 → "N장으로 입차 등록" | **업로드 성공** (진행률 바 done 증가, failed=0) |
| 1-3 | 업로드 후 ticket 확인 (DB `mepark_tickets.vehicle_photos`) | 경로 배열 저장됨 `{org_id}/{ticket_id}/0_front.jpg` |
| 1-4 | "사진 없이 입차" (0장 패스) | 정상 입차, PATCH 생략/빈배열 OK |

**🚨 1-2에서 막히면 (403 / policy violation / "new row violates row-level security"):**
→ Supabase SQL Editor에서 `sql/storage-vehicle-photos.sql` 실행 → 재시도.
실행 후 검증 쿼리로 정책 4종(`%vehicle photos%`) 생성 확인.

**⚠️ 별도 갭(검증 아님, 기록용):** v2에 vehicle_photos **표시 뷰어 없음**. 업로드/저장만 되고 보는 화면이 없어 "사고 증거"로서 가치 미완 → P1 외 후속 작업 필요.
